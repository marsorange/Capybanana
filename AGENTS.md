# Capybanana — project orientation

竖屏 Web 随心陪伴养成小游戏。核心气质是“每天一分钟，陪低边形卡皮巴拉长大一点”：短互动、轻养成、卡片收藏、移动端优先。MVP 仍保持全程序化低多边形 3D，无外部美术素材、无 AI/图片 API。

- **状态/路由**：`src/state/gameStore.ts`（Zustand + persist，`skipHydration`，客户端 `rehydrate()`）。屏幕用 `screen` 状态机切换，不用路由。登录后客户端只投影云端 `CloudSave`，不再保留访客/本地自动结算玩法；`GameRoot.tsx` 通过 `cloudPull()` 轮询服务端状态。
- **生命周期**：`idle_home → ready(行李已备) → traveling → idle_home`。纯逻辑在 `src/game/clock.ts`；`ready` 不会自动出门，必须由 Agent 调 `/api/agent/day` 决定。`advanceLifecycle` 只负责把已经 `traveling` 的云端行程在到时后结算归来。
- **内容/规则**：`src/game/destinations.ts`（10 主题文案 + 关键词规则）、`planTrip.ts`（加权随机选目的地，对玩家隐藏）、`generatePostcard.ts`（按留言/行李/性格拼装文案）。
- **3D（只有两个模型：home + 主角）**：`src/components/scenes3d/` 故意收敛成「两个模型 + 舞台/系统」。**对外只有两个模型入口**：`home/HomeModel.tsx`（整个家场景）和 `character/CharacterModel.tsx`（主角）。其余都是支撑层，不是「模型资产」：根目录 `SceneCanvas`(相机/灯光舞台)、`materials`(共享 toon 材质)、`zoomBus`、`RoamingCompanion.tsx`(把主角放进 home 场景 + 寻路的胶水)。统一走 `materials.ts` 的 **`toonMaterial(color, opts?)`** 工厂（缓存版 `MeshToonMaterial` + gradientMap cel ramp + Fresnel 边缘光 + 注入式「星球弯曲」顶点位移）+ drei `Outlines`；各 part 文件原来的 `m()`/`glow()` 助手现在都委托成 `<primitive object={toonMaterial(c)} attach="material" />`（共享实例，N 个同色网格共用一个已编译程序）。**坑**：别在 JSX 里给 `<meshToonMaterial>` 传 `flatShading`（R3F 类型不支持，会编译失败）——工厂在 TS 里 `new MeshToonMaterial(...)` 后用窄 cast 设 `flatShading`，绕开它。「星球弯曲」是 `sceneBend` 共享 uniform，由 `SceneCanvas` 的 `bendStrength` 开关（home 场景给 0.15、转盘屏默认 0=不弯）：**只弯视觉网格、碰撞体不弯**，宠物走的中心区 bend≈0 所以两边不打架。**加新模型/换 3D 时只改这两个入口，别再往外散文件。**
- **主角模型（`character/CharacterModel.tsx`，单文件）**：一个组件搞定——按 `type` 解析物种再画（已内含 `normalizeSpecies`，旧类型也稳）。**没有 dispatcher/registry**（之前的 `Character3D`/`models.ts` 已删）。物种就是 `CompanionType` = `capybara|rabbit|duck|raccoon|shiba|sheep`，花名册唯一真源在 `src/game/characters.ts`（`CHARACTERS`：每个角色的中文名/默认色/招牌配饰/耳朵比例/参考图名 + `normalizeSpecies` 旧存档兜底映射）。参考美术图在 `src/asset/Character/*.png`（Capybanana/Bunberry/Quackaroo/Raccoonie/Shibuddy/Woollybean）。**3D 资产管线已就位（双路径，仍单文件）**：`character/companionModels.ts` 是「物种 → GLB」清单（数据，不是文件 registry）；列入的物种走 `CharacterModel` 的 **GLB 路径**（`useGLTF` 加载 → 克隆 → 换上 `toonMaterial(color)` → 按包围盒**自动归一化**到统一身高 + 脚踩地 y=0，所以任意 Tripo 导出都能直接对齐程序化体的取景），没列的退回**程序化占位体**（6 物种共享、按 type/color/耳朵比例参数化、大眼 + 腮红）。两条路径 props 完全一致，**加新资产 = 清单加一行**，零调用方改动。**运行时 GLB 必须放 `public/models/`**（`src/asset/3D/` 只是源、Next 运行时取不到）；Draco 解码器在 `public/draco/`（清单的 `COMPANION_DRACO_PATH`，未压缩 GLB 不会真去取它）。当前 `capybara` 已接 Tripo 生成的占位 GLB（单网格 → 单色 tint，多色细节不如程序化版——是资产本身的取舍，不是管线问题）。`labels.ts` 的 `COMPANION_TYPES`/`TYPE_LABELS`、`postcardPrompt` 的物种描述都从花名册派生。**换形象**：新账号默认领养随机一个角色（`randomCuteCompanion`：从 6 角色里随机挑一个，连默认色+招牌配饰一起套，读起来就是那只参考角色）；已有宠物改外观走 `engine.restyleCompanion`（只改 type/color/accessory，不动名字/数值/历史），入口三个——`POST /api/agent/restyle`、ProfileScreen 的「🎲 换个样子」按钮、测试链接 `GET /agent/restyle?bind=<token>`。`coerce*` 现接受任意 `#rrggbb` 颜色。
- **家模型（`home/HomeModel.tsx`，单入口）**：等距 diorama = `Island`(浮空草地小岛) + `House`(两层剖面别墅，向 +x/+z 开切口) + `Yard`(院子：栅栏/踏石/树/菜地/花)。这三块几何是**私有部件**，放在 `home/parts/`（含 `Backpack.tsx`，PackScreen 也复用它），外部一律只 import `HomeModel`。配套：`home/layout.ts`(原 `villaLayout`，SPOTS/楼梯等寻路数据)、`home/interaction/`(`InteractionLayer`/`InteractionMarker`/`commandBus`/`navBus`)。镜头是**正交等距 + 双指缩放**（`SceneCanvas` 的 `orthographic`/`enableZoom`/`azimuth`/`minZoom`..`maxZoom`）。**注意：当前 home 美术是占位**，会整体重做——别在这堆装饰几何上花功夫优化（instancing 等），等新美术定稿。
- **物理框架（@react-three/rapier = Rapier 的 Rust→WASM 物理）**：目标是「整页真实物理手感」。`SceneCanvas` 加了 `physics` 开关（包一层 `<Physics gravity debug>`，只有 home 场景开；人像转盘屏不开）。**核心铁律：渲染网格 ≠ 碰撞体**——HomeModel 那几百个装饰网格全是纯视觉，物理世界只认 `home/HomeColliders.tsx` 里那十来个隐形 fixed collider（地面/外墙/两面实墙，尺寸照 `layout.ts` 近似；美术重做时换视觉、碰撞体契约不变）。HomeColliders 的隐形地面 plane 还兼「点地走」拾取器（写 `navBus`）。主角 `RoamingCompanion.tsx` 已是 **Rapier `KinematicCharacterController`**：kinematic 刚体 + 胶囊 collider，每帧「朝目标的水平位移 + 重力」交给 controller 解算（贴地、爬小台阶、贴墙滑行不穿模）。**无用户操控**，目标只来自三种——点地(`navBus`) / 场景 marker(`commandBus`，含日记入口回调) / 自动在 floor-0 的 SPOTS 间漫游；楼梯/上下楼暂缓（等美术加真坡道）。物理道具样板 `home/PhysicsToy.tsx`（dynamic `<RigidBody>` 小球，点一下抛出会弹会滚）——以后背包物品掉落、积木堆叠都照这个抄。新 3D：**静态装饰**用 `materials.ts` 的 `toonMaterial(color)`（cel + 弯曲 + 边缘光）；**动态物理道具**用 `flatMaterial(color)`（缓存版 Lambert，刻意不进 toon/不进星球弯曲——滚动的球不该被掰弯，且省一个 shader 给物理腾余量）。坑：磁盘上有两份 `@dimforge/rapier3d-compat`（顶层 0.12 仅 `@types/three` 类型用，运行时走 @react-three/rapier 的 0.19），所以别裸 `import` 它的类型，要从 `world.createCharacterController` 的返回类型推。
- **打包（已极简）**：`PackScreen` 只有一个放大的 3D 背包 + 📷拍照 + 💬留言 + 确认，无预设网格、无快捷标签。可拍多张，每拍一张确认后自动进背包（`photos: PackedItem[]` 缩略图列，可单独移除）。每样 `PackedItem`（`preset` 仅 Agent API 用 / `photo`）。`CameraCapture.tsx` 拍完先用 `photoExtract.ts` 取主色得“氛围”标签（`tags` 仍按颜色，喂 outcome），再 `POST /api/recognize` 调 MiniMax 视觉理解（`src/lib/minimax.ts` 的 `understandImage`→Token Plan 视觉端点 `/v1/coding_plan/vlm`，复用 `MINIMAX_API_KEY`）得“这是什么”，把物体名填进 `hint`/`keyword`/`label`；没配 key 或识别失败则保留取色启发式，不打断。摄像头需安全上下文（localhost/https）。
- **账号 / Agent 连接（云存档）**：**只用 Supabase Auth 登录（Google）**，已移除手机号登录与访客「本地玩」入口——未登录（`!cloud`）只能看到登录页。流程是 **bridge 模式**：浏览器用 `@supabase/supabase-js`（`src/lib/supabaseClient.ts` 的 `signInWithGoogle`）走 Google OAuth，登录后 `GameRoot` 的 `useSupabaseAuthBridge` 监听到 Supabase session，把 `access_token` POST 给 `POST /api/auth/supabase`；服务端 `supabase.auth.getUser(token)` 验证身份，按 Supabase user id 在 `store.loginBySupabase` 找/建账号，铸出我们自己的 `bindToken`。云存档 **SQL-only**，只认 `POSTGRES_URL`/`POSTGRES_PRISMA_URL`/`POSTGRES_URL_NON_POOLING`，需先执行 `supabase/migrations/0001_storage_refactor.sql`；没有 PostgreSQL 时登录 route 直接 503。AI Agent 读 `skill.md` 后用 `/api/agent/*` 操作同一只云宠物；网页端只是持有同一个 bind token 的瘦客户端。
- **Agent 驱动的玩法（核心）**：云宠物**不再自己出门**——主人在网页备好「今日包裹」（物品 + 心愿），宠物停在 `ready` 待命，由 **Agent** 看状态 + 包裹后调用统一决策接口 `POST /api/agent/day`：`{action:"travel", destination?, note?}` 触发旅行，归来寄明信片；`{action:"stay", mode?, note?}` 立即结算在家/院子/休息。当前 Claw 对战玩法已后置：没有 `/api/agent/battle`、`BattleRecord`、`CloudSave.battles` 或相册对战页；`action:"battle"` 会被拒。明信片 AI 生图缓存也已下线，未来如果恢复，应把图片/状态存到 Supabase，不再引入 KV。
- **部署（Vercel）**：所有 route 文件均 `runtime="nodejs"`+`dynamic="force-dynamic"`。云存档需要 Supabase/Postgres，配置 `POSTGRES_URL`（优先 pooled）并运行迁移；不再支持 Upstash/Vercel KV 或进程内存作为账号存储。`/agent/skill.md` 用点号文件夹路由，绑定链接域名从请求头推导，可用 `APP_BASE_URL` 固定。
- **UI 主题**：Tailwind v4 `@theme` in `globals.css`（`bg-cream`/`text-ink`/`bg-accent #D95F59`/`rounded-sticker`/`font-hand`）。标题字体用系统 CJK（Yuanti/PingFang），不外链 Google Fonts。`ErrorBoundary.tsx` 包住每屏，3D 出错只显示重试卡片不至于全白。
- **验证 & 坑**：dev 运行时用 `npx tsc --noEmit`，**不要在 `next dev` 运行时跑 `next build`**（两者共用 `.next`，会让 dev 加载不稳定/白屏；修复：`rm -rf .next` 后重启 dev）。
- 仓库目录名 `Capybanana` 含大写，npm 包名不能含大写——脚手架曾用小写临时子目录再上移。
- **远期玩法蓝图（用户已提出，尚未实现）**：纪念品/特产、负伤与休养、成长、对战与战后恢复、家庭活动、按“天”推进的日循环；在家自主生活由 时间/心情/体力/是否受伤/纪念品 驱动选择活动。当前只实现到“出门旅行→寄明信片→带回”闭环。
