# Capybanana — project orientation

竖屏 Web 随心陪伴养成小游戏。核心气质是“每天一分钟，陪低边形卡皮巴拉长大一点”：短互动、轻养成、卡片收藏、移动端优先。主体验仍保持低多边形 3D；GLB 管线暂时保留但默认关闭，明信片 AI 生图仍下线。

- **状态/路由**：`src/state/gameStore.ts`（Zustand + persist，`skipHydration`，客户端 `rehydrate()`）。屏幕用 `screen` 状态机切换，不用路由。登录后客户端只投影云端 `CloudSave`，不再保留访客/本地自动结算玩法；`GameRoot.tsx` 通过 `cloudPull()` 轮询服务端状态。
- **生命周期**：`idle_home → ready(行李已备) → traveling → idle_home`。纯逻辑在 `src/game/clock.ts`；`ready` 不会自动出门，必须由 Agent 调 `/api/agent/day` 决定。`advanceLifecycle` 只负责把已经 `traveling` 的云端行程在到时后结算归来。
- **内容/规则**：`src/game/destinations.ts`（**8 个目的地，分 near/far 两池** + 关键词规则 + 每地 3 档稀有度地标 `LANDMARKS`）、`planTrip.ts`（`pickDestination`：在 Agent 选定的 near/far 池里、按打包关键词**加权随机**选目的地，对玩家隐藏）、`gacha.ts`（3 档稀有度 N/R/SR、24 张图鉴、服务端掷骰，**稀有度纯随机、不受打包影响**）、`generatePostcard.ts`（程序化拼装文案，作为 LLM 无 key 时的回退）。**明信片文案在归来时由 LLM 写**：`server/llm/postcard.ts` 的 `composePostcard` 拿到已定死的稀有度+地标，融合打包物品/留言/Agent note 写文案；`engine.dressTravelPostcard` 统一钉上地标 + 文末「寄自」签名（卡面与文字必一致）。
- **3D（只有两个模型：home + 主角）**：`src/components/scenes3d/` 故意收敛成「两个模型 + 舞台/系统」。**对外只有两个模型入口**：`home/HomeModel.tsx`（整个家场景）和 `character/CharacterModel.tsx`（主角）。其余都是支撑层，不是「模型资产」：根目录 `SceneCanvas`(相机/灯光舞台)、`materials`(共享 toon 材质)、`zoomBus`、`RoamingCompanion.tsx`(把主角放进 home 场景 + 寻路的胶水)。统一走 `materials.ts` 的 **`toonMaterial(color, opts?)`** 工厂；各 part 文件的 `m()`/`glow()` 助手都委托成 `<primitive object={toonMaterial(c)} attach="material" />`（共享实例，N 个同色网格共用一个已编译程序）。**2026-06-12 渲染重做（回归 toon，根治白边）**：`toonMaterial` 现在是 **`MeshToonMaterial` + 共享 32px「软 cel」shaped ramp（Linear，阴影平台→明暗交界硬拐点→亮部缓坡）+ flatShading**——flat 面每面吃一个恒定色调，亮部缓坡让向阳面散开成一片临近色阶（这就是低多边形的「质感」，零开销）；曾试过 3 阶 Nearest 硬色阶，所有面压成三块平色、被反馈「缺质感」——之前的软 PBR（哑光 MeshStandardMaterial + SoftEnv 程序化 IBL）+ `PostFX.tsx` 后处理链（N8AO/SMAA/TiltShift/调色）**整套已删**，`@react-three/postprocessing` 依赖也已移除。原因：透明画布上 EffectComposer 的模糊/AA 会把场景 RGB 泄进 alpha=0 像素，浏览器预乘合成读成贴轮廓的白色光晕（删 Bloom 后白边仍在的根因）；且 toon 是 three 最便宜的受光着色器、单 forward pass，手机更凉。光照配方（SkyWeather/SceneCanvas 静态分支同款）：**一盏强方向光 key + 弱 hemisphere 环境光 + 一盏微弱冷 fill**（key:ambient ≈ 2.5:1，强度按 three 物理光标尺 ÷π ≈ 经典 toon 的 key 1.2 / ambient 0.45），色阶带子全靠这个比例立起来。AA 回画布 MSAA（`antialias: true`），色调映射用 R3F 默认的 renderer ACES（不再有 ToneMapSync）。保留的性能纪律：30fps 限帧（`fpsCap`，`frameloop="demand"` + rAF 节流，后台自动停渲）、dpr cap、`sun` 开启时不挂 ContactShadows、1024 阴影图 + 紧 frustum + PCF、context-loss 螺旋断路器。**坑**：别在 JSX 里给材质传 `flatShading`（R3F/three 类型不支持）——工厂 `new MeshToonMaterial(...)` 后用窄 cast 设；窗玻璃仍是近透明非自发光（opacity 0.16），背后 DOM 层天空图透出来。**星球弯曲着色器已删**（`sceneBend`/`bendStrength` 不存在了，删前所有调用方都已是 0）——以后要小星球感就改真实半球地面几何，别再写顶点位移 patch。**加新模型/换 3D 时只改这两个入口，别再往外散文件。**
